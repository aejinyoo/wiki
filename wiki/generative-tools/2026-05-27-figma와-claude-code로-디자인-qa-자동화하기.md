## 한국어 요지

이 게시물은 Figma와 Claude Code를 연동하여 디자인 QA 및 수정 과정을 자동화하는 방법을 상세히 안내합니다. 먼저, Claude Code 플러그인을 Figma에 설치하는 방법(맥에서는 'sudo' 명령어 사용)을 설명합니다. 다음으로, 토큰 하드코딩 감지, Story 누락 경고, 접근성 검사 등 QA 규칙을 Claude Code에 심어 자동화된 에이전트와 훅을 설정하는 QA 하네스 생성 단계를 소개합니다. "QA 돌려줘"라는 간단한 요청만으로 빌드, 토큰 준수, Story 완전성, Figma 동기화, 접근성까지 자동으로 검사할 수 있습니다. QA에서 발견된 위반 항목은 코드로 직접 수정하며, 하드코딩된 색상 토큰을 교체하거나 누락된 Story를 생성하고 접근성 이슈를 해결하는 과정을 포함합니다. 또한, 수정 전후 비교 리포트를 엑셀로 자동 생성하여 위반 수, 해결율, 미해결 항목 및 수동 수정 가이드까지 제공합니다. 마지막으로, 코드에서 발견된 토큰 불일치를 Figma Variables에도 반영하여 디자인과 코드 간의 양방향 일치를 유지하는 방법을 설명합니다. 초기 설정의 어려움에도 불구하고, 한 번 구축하면 디자인 시스템 관리의 효율성을 크게 높일 수 있다고 강조합니다.
